About

Hi! My name's Brandon, and I'm a Software Engineer who's been programming for over eight years. I am currently studying Computer Science and Engineering at UCLA. I specialize in low-level programming, particularly Game Engines and Embedded Systems.

Professionally, I worked as a Firmware Engineer at a security camera startup called Rhombus Systems and developed a brand new device to test and debug cameras over PoE. I created a firmware from scratch based on Linux to run on a small 580 MHz CPU that could boot up in seconds and was less than 8 Mb in size.

In 2020, I created an app with the help of some other developers to allow for iMessage texts to be sent and received on non-Apple devices. I worked on the Android side of things to create a client in Flutter that could communicate with our Mac server to send and receive texts. The client also had to manage an SQLite database of messages synchronized with the server.

Computer Graphics and Game Engine development are what I enjoy most. I wrote my own custom 3D Game Engine in C++ using Vulkan with support for PBR and a full blown editor. It is fascinating to see how modern AAA games have to run multiple render passes with thousands of polygons all in under 16 ms. I'm currently working on a new Game Engine with a professional Game Developer with the goal of making all of my future games in it!

Personal Projects

Custom 3D Game Engine

C++ | Vulkan

Vultr is a custom 3D Game Engine built from scratch. It uses a Vulkan rendering backend with an editor that uses ImGui. It supports run-time reflection in the editor, gameplay hot-reloading, an ECS system, optimized asset imports, and a complete material system. The engine can render PBR graphics with shaders for roughness, metalness, normal, and albedo. The meshes, textures, shaders, and all other assets are loaded using separate thread submission queues to maximize performance. It also uses a minimal subset of the C++ STL, opting for a custom template library where possible.
